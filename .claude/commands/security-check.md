# セキュリティクイックチェック

現在のコードに対して **MVP 必須のセキュリティ要件** を満たしているか確認してください。

## チェック項目

### 1. マルチテナント分離（RLS）

- `src/` 内の全 Supabase クエリを確認し、`tenant_id` フィルタが漏れている箇所を探す
- `supabase/migrations/` に RLS ポリシーが全テーブルに存在するか確認する
- `SUPABASE_SERVICE_ROLE_KEY` を使うクエリは必ず `tenant_id` を WHERE 句で指定しているか

### 2. 環境変数の露出

- `NEXT_PUBLIC_` プレフィックスが付いた変数が `NEXT_PUBLIC_SUPABASE_URL` と `NEXT_PUBLIC_SUPABASE_ANON_KEY` 以外に存在しないか確認する
- `src/app/` 内のクライアントコンポーネントで API キーを直接使っていないか

### 3. 個人情報・データ保護

- メールアドレス・氏名・電話番号などの個人情報がログに出力されていないか
- CSV取込（F-02B）でファイルサイズ・MIMEタイプのバリデーションが実装されているか

### 4. 配信停止管理（特定電子メール法）

- 全配信メールに配信停止リンクが自動付与されているか（F-06A）
- 配信停止リクエスト受信後、`unsubscribe_list` に即座に追加されているか
- 停止済みアドレスへの再配信防止ロジックが実装されているか

### 5. API Routes の認証

- `src/app/api/` 配下の全エンドポイントで Supabase Auth のセッション検証が行われているか
- 未認証リクエストが 401 を返すか

## 報告形式

問題を発見した場合は、**ファイルパス:行番号** を明示し、修正案とともに報告してください。
問題がなければ「✅ セキュリティチェック通過」と報告してください。

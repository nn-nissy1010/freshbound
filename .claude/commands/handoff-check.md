# 引き継ぎ準備チェック（社内エンジニア向け）

Week 7 の引き継ぎに向けて、**納品物の充足状況**を確認してください。

## チェックリスト

### ドキュメント（リポジトリ内に存在するか確認）

- [ ] `README.md` — プロジェクト概要・ローカル起動手順が記載されているか
- [ ] アーキテクチャ図（`docs/architecture.md` 等）— システム構成・コンポーネント関係
- [ ] DB設計書（`docs/database.md` 等）— テーブル定義・ER図・RLS設計
- [ ] API仕様書（`docs/api.md` 等）— 全エンドポイントのリクエスト/レスポンス例
- [ ] 環境構築手順書（`docs/setup.md` 等）
- [ ] デプロイ手順書（`docs/deploy.md` 等）
- [ ] 運用マニュアル（`docs/operations.md` 等）
- [ ] 既知の不具合一覧（`docs/known-issues.md` 等）
- [ ] Phase 2 拡張ポイント（`docs/phase2-extension.md` 等）

### コード品質

- [ ] `npm run test:coverage` でカバレッジ 50% 以上か
- [ ] `npm run lint` がエラー 0 か
- [ ] `npx tsc --noEmit` がエラー 0 か
- [ ] 複雑なロジックに日本語コメントがあるか

### セキュリティ

- [ ] 全テーブルに RLS ポリシーが設定されているか（`supabase/migrations/` 確認）
- [ ] `.env.local` が `.gitignore` されているか
- [ ] `NEXT_PUBLIC_` な変数がサーバー専用の機密情報を含んでいないか
- [ ] F-06A（配信停止管理）が特定電子メール法に準拠して実装されているか

### Phase 2 への橋渡し

- [ ] ステップメール（F-10 Phase 2）の組み込み位置がコメントで示されているか
- [ ] 匿名トラッキング（F-13）のための URL パラメータが配信メールに設計されているか
- [ ] Inngest の追加手順がドキュメントに記載されているか

## 報告形式

不足している項目を箇条書きでリストアップし、優先度（高/中/低）をつけて報告してください。

# MVP スコープチェック

現在実装中のコード・設計が **MVP として問題ないか** を確認してください。

## 判定の基本方針

「設計書の機能リストに完全一致するか」ではなく、**「明らかにMVPではないものが混入していないか」** を判定してください。
MVP機能を実現するために必要な実装は、設計書に細部が明記されていなくても許容します。

---

## ✅ MVP スコープ内（実装してよいもの）

### 顧客向け機能（F-01〜F-10）

- F-01 ICP設計（簡易版）
- F-02 / F-02A / F-02B 企業発掘・新規HP判定・CSV取込
- F-03 / F-03A 担当者発掘・電話番号取得
- F-04 AIスコアリング（簡易版・業種/規模/地域の3軸）
- F-05 AIメール生成（GPT-4o-mini）
- F-06 / F-06A SendGrid配信・配信停止管理（特定電子メール法対応）
- F-07 反応検知（開封/返信）
- F-08 ホットリード通知（LINE）
- F-09 ダッシュボード（基本）
- F-10 マルチテナント・RLS・Supabase Auth・代理店（agencies）管理

### 管理者パネル（FA-01〜FA-10） — Control Tower

- FA-01 管理者ダッシュボード
- FA-02 テナント管理（顧客一覧）
- FA-03 代理店管理
- FA-04 ユーザー管理（全テナント横断）
- FA-05 配信監視（キャンペーン・メールログ・配信停止）
- FA-06 CSV管理
- FA-07 API・外部サービス監視（SendGrid/OpenAI/musubu等）
- FA-08 分析（MRR・テナント数・開封率等）
- FA-09 システム設定（SMTP/SendGrid/OpenAI/キュー/セキュリティ/通知/フィーチャーフラグ）
- FA-10 監査ログ

### 請求・決済（FB-01〜FB-02） — Stripe連携

- FB-01 Stripe課金連携（`stripe_customer_id`・`stripe_subscription_id`・Webhook）
- FB-02 請求書管理（`invoices`テーブル）

### 許容される付随実装

- 上記機能を動かすために必要なユーティリティ・共通コンポーネント・型定義・ミドルウェア
- 上記機能の「簡易版」として実装する場合、設計書より機能が少なくても問題なし
- `queue_emails`・`delivery_logs`・`failed_logs`・`generated_email_recipients` 等のインフラテーブル

## 判定結果の報告

- ✅ **問題なし**：MVP 機能の実装として妥当であることを確認、作業継続を推奨
- ⚠️ **要確認**：グレーゾーンの実装を検知した場合、理由を説明した上で「許容 / 削除 / 延期」の選択肢を提示
- 🚫 **ブロック**：Phase 2/3 機能の混入を検知した場合、具体的なコード箇所を示して除去を推奨
